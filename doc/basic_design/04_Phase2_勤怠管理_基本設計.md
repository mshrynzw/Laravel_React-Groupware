## 1. 目的・スコープ

従業員の出勤・退勤を記録し、履歴参照および日別・月別の集計を提供する。打刻 UI と一覧画面を含む。

---

## 2. 機能概要

| 項目 | 内容 |
|------|------|
| attendance_records テーブル | ユーザー、日時、種別（出勤／退勤等） |
| 打刻 API | 出勤／退勤の記録、重複・不正打刻の防止ルール |
| 勤務履歴取得 API | 期間指定、ユーザー指定 |
| 日別・月別集計ロジック | 労働時間、日次の整合性 |
| 打刻 UI | 当日の打刻ボタン、状態表示 |
| 勤怠一覧画面 | 月次カレンダーまたは表形式 |

---

## 3. 画面・API（概要）

### 3.1 画面

- 打刻画面
- 勤怠一覧（自分／管理者は全体など権限で切替）

### 3.2 API（例）

| メソッド | パス（案） | 説明 |
|----------|------------|------|
| POST | `/api/attendance/clock-in` | 出勤打刻 |
| POST | `/api/attendance/clock-out` | 退勤打刻 |
| GET | `/api/attendance/records` | 履歴取得 |
| GET | `/api/attendance/summary` | 日別・月別集計 |

---

## 4. データモデル（案）

| テーブル | 主な項目 |
|----------|----------|
| `attendance_records` | id, user_id, type, recorded_at, … |

---

## 5. 非機能・業務ルール

- タイムゾーン（日本時間）の統一。
- 同日複数出勤の扱い、修正フローは要件に応じて別途。
- **監査ログ**: 打刻および勤怠データの修正（導入する場合）について、**誰が・いつ・どのレコードに対して・何を変更したか**を記録する。

---

## 6. 依存関係

- **前提**: 認証済みユーザー ID。
- **連携**: 給与計算（勤怠データ集計）、ダッシュボード「今日の勤怠」。
