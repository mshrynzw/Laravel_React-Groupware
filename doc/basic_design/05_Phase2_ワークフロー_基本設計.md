---

## 1. 目的・スコープ

有給等の申請をデータとして保持し、承認者による承認・却下およびフロー制御を行う。申請内容の可変部分は JSONB で保持する想定。

---

## 2. 機能概要

| 項目 | 内容 |
|------|------|
| requests テーブル（JSONB） | 申請種別、ペイロード、状態、申請者 |
| approvals テーブル | 申請 ID、承認者、承認順、結果、日時 |
| 申請作成 API | 下書き／提出の区別は要件次第 |
| 承認 API | 承認・却下、次承認者への進行 |
| 承認フロー制御ロジック | 多段承認、代理承認はスコープ外なら記載 |
| 申請一覧画面 | 自分の申請／承認待ちの切替 |
| 承認 UI | 詳細表示と承認アクション |

---

## 3. 画面・API（概要）

### 3.1 画面

- 申請一覧・詳細・新規申請
- 承認待ち一覧・承認画面

### 3.2 API（例）

| メソッド | パス（案） | 説明 |
|----------|------------|------|
| GET | `/api/requests` | 一覧 |
| POST | `/api/requests` | 作成 |
| GET | `/api/requests/{id}` | 詳細 |
| POST | `/api/requests/{id}/approve` | 承認 |
| POST | `/api/requests/{id}/reject` | 却下 |

---

## 4. データモデル（案）

| テーブル | 主な項目 |
|----------|----------|
| `requests` | id, user_id, type, payload (JSONB), status, … |
| `approvals` | id, request_id, approver_user_id, step_order, result, … |

---

## 5. 非機能

- JSONB のスキーマバージョン管理（申請種別ごと）を検討。
- **監査ログ（操作履歴）**: 申請の**作成・更新・提出・取下**、**ステータス変更**、**承認・却下**、payload（JSONB）の**重要な変更**について、**誰が・いつ・どの申請に対して・何を変更したか**を追跡できるよう記録する（承認操作に限らず、業務上の状態遷移をすべて対象とする）。

---

## 6. 依存関係

- **前提**: ユーザー・組織・承認者の関係が定義できること。
- **連携**: ダッシュボード「未承認申請」、勤怠（有給日数）と連携する場合は別設計。
