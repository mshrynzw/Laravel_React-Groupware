## 1. 目的・スコープ

組織内向けの告知を投稿・閲覧する。管理者による投稿と、一般ユーザー向けの一覧・詳細表示を対象とする。

---

## 2. 機能概要

| 項目 | 内容 |
|------|------|
| announcements テーブル | タイトル、本文、公開日時、投稿者、公開範囲（任意） |
| 投稿 API | 管理者のみ実行可 |
| 一覧取得 API | ページング、新着順 |
| 詳細 API | ID 指定 |
| 管理者投稿画面 | 作成・編集・公開 |
| 一覧 UI | トップや専用ページでの一覧 |

---

## 3. 画面・API（概要）

### 3.1 画面

- お知らせ一覧
- お知らせ詳細
- 管理：投稿・編集

### 3.2 API（例）

| メソッド | パス（案） | 説明 |
|----------|------------|------|
| GET | `/api/announcements` | 一覧 |
| GET | `/api/announcements/{id}` | 詳細 |
| POST | `/api/announcements` | 作成（管理者） |
| PUT/PATCH | `/api/announcements/{id}` | 更新 |
| DELETE | `/api/announcements/{id}` | 削除 |

---

## 4. データモデル（案）

| テーブル | 主な項目 |
|----------|----------|
| `announcements` | id, title, body, published_at, author_user_id, … |

---

## 5. 非機能

- XSS 対策（本文が HTML の場合はサニタイズ）。
- 公開前下書きの要否は要件で決定。
- **監査ログ**: お知らせの作成・更新・削除・公開／下書きの切替について、**誰が・いつ・どのお知らせに対して・何を変更したか**を記録する。

---

## 6. 依存関係

- **前提**: 認証・RBAC（管理者ロール）。
- **連携**: ダッシュボード「最新のお知らせ」。
