## 1. 目的・スコープ

社内でファイルをアップロード・保存し、権限のあるユーザーがダウンロード・一覧参照できるようにする。保存先は S3 またはローカルストレージを選択可能とする。

---

## 2. 機能概要

| 項目 | 内容 |
|------|------|
| files テーブル | 論理名、保存パス／キー、サイズ、MIME、所有者、公開範囲 |
| S3 or ローカル保存設計 | ドライバ切替、署名付き URL またはプロキシダウンロード |
| アップロード API | マルチパート、サイズ上限、ウイルススキャンは任意 |
| ダウンロード API | 認可後にストリームまたはリダイレクト |
| ファイル一覧 UI | フォルダ階層の要否は要件次第 |

---

## 3. 画面・API（概要）

### 3.1 画面

- ファイル一覧
- アップロード操作

### 3.2 API（例）

| メソッド | パス（案） | 説明 |
|----------|------------|------|
| GET | `/api/files` | 一覧 |
| POST | `/api/files` | アップロード |
| GET | `/api/files/{id}/download` | ダウンロード |

---

## 4. データモデル（案）

| テーブル | 主な項目 |
|----------|----------|
| `files` | id, user_id, original_name, storage_disk, path_or_key, size, mime_type, … |

---

## 5. 非機能・セキュリティ

- アクセス制御（本人・グループ・全体）。
- `.env` にバケット名・リージョン等を保持し秘匿。
- **監査ログ**: ファイルのアップロード・削除、共有範囲・権限の変更について、**誰が・いつ・どのファイルに対して・何を変更したか**を記録する。

---

## 6. 依存関係

- **前提**: 認証・ユーザー ID。
- **インフラ**: AWS S3 利用時は IAM 権限の設計。
