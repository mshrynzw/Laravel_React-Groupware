## 1. 目的・スコープ

ルーム単位でメッセージを送受信し、WebSocket によりリアルタイムに画面へ反映する。

---

## 2. 機能概要

| 項目 | 内容 |
|------|------|
| chat_rooms テーブル | ルーム名、種別（直接／グループ等） |
| chat_messages テーブル | room_id, user_id, body, 送信日時 |
| メッセージ送信 API | HTTP での送信＋ブロードキャスト |
| WebSocket 導入 | Laravel Reverb 等、チャネル認可 |
| リアルタイム更新 | フロントでイベント購読 |
| チャット UI | ルーム一覧、メッセージスレッド |

---

## 3. 画面・API（概要）

### 3.1 画面

- チャットルーム一覧
- メッセージ画面

### 3.2 API・イベント（例）

| 種別 | パス／チャネル（案） | 説明 |
|------|----------------------|------|
| GET | `/api/chat/rooms` | ルーム一覧 |
| POST | `/api/chat/rooms/{id}/messages` | メッセージ投稿 |
| WS | `private-chat.{roomId}` 等 | 新着配信 |

---

## 4. データモデル（案）

| テーブル | 主な項目 |
|----------|----------|
| `chat_rooms` | id, name, … |
| `chat_messages` | id, chat_room_id, user_id, body, created_at, … |

ルーム参加者テーブルが必要な場合は別途定義。

---

## 5. 非機能

- メッセージのページング・無限スクロール。
- 接続数・サーバリソース（Reverb／ワーカー）。
- **監査ログ**: メッセージの送信および（あれば）編集・削除、ルームの作成・参加・退出について、**誰が・いつ・どのリソースに対して・何を行ったか**を追跡できるよう記録する。

---

## 6. 依存関係

- **前提**: 認証（WebSocket でも同一ユーザーを識別）。
- **インフラ**: 本番での WSS 終端、スケール時のブロードキャスト設定。
