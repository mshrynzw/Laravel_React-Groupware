## 1. 目的・スコープ

タスクの登録・更新・削除とステータス管理、担当者の紐付けを行う。カンバン UI での可視化を対象とする。

---

## 2. 機能概要

| 項目 | 内容 |
|------|------|
| tasks テーブル | タイトル、説明、ステータス、期限、作成者 |
| タスク CRUD API | REST 形式 |
| ステータス管理 | 未着手／進行中／完了等、ワークフローは固定値または設定化 |
| 担当者紐付け | assignee_user_id、複数担当は要件次第 |
| カンバン UI | 列＝ステータス、ドラッグでステータス更新 |

---

## 3. 画面・API（概要）

### 3.1 画面

- カンバンボード
- タスク詳細モーダル／ページ

### 3.2 API（例）

| メソッド | パス（案） | 説明 |
|----------|------------|------|
| GET | `/api/tasks` | 一覧（フィルタ） |
| POST | `/api/tasks` | 作成 |
| GET | `/api/tasks/{id}` | 詳細 |
| PUT/PATCH | `/api/tasks/{id}` | 更新（ステータス含む） |
| DELETE | `/api/tasks/{id}` | 削除 |

---

## 4. データモデル（案）

| テーブル | 主な項目 |
|----------|----------|
| `tasks` | id, title, description, status, due_at, creator_user_id, assignee_user_id, … |

---

## 5. 非機能

- 同時編集の競合（楽観ロックまたは最終更新優先）を検討。
- **監査ログ**: タスクの作成・更新・削除、担当者・ステータス・期限の変更について、**誰が・いつ・どのタスクに対して・何を変更したか**を記録する。

---

## 6. 依存関係

- **前提**: ユーザー一覧から担当者を選択可能であること。
