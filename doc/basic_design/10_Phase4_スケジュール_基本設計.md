## 1. 目的・スコープ

予定（イベント）の登録・編集・削除と、カレンダー上での月／週表示を提供する。

---

## 2. 機能概要

| 項目 | 内容 |
|------|------|
| schedules テーブル（または events） | タイトル、開始・終了、終日フラグ、ユーザー、繰り返しは任意 |
| イベント CRUD API | 期間指定での一覧取得を含む |
| カレンダー表示ロジック | 月ビュー・週ビュー用のクエリ |
| 月／週表示 UI | ライブラリ選定は実装フェーズで決定 |

---

## 3. 画面・API（概要）

### 3.1 画面

- カレンダー（月／週切替）
- イベント作成・編集フォーム

### 3.2 API（例）

| メソッド | パス（案） | 説明 |
|----------|------------|------|
| GET | `/api/schedules` | `from`/`to` で期間取得 |
| POST | `/api/schedules` | 作成 |
| GET | `/api/schedules/{id}` | 詳細 |
| PUT/PATCH | `/api/schedules/{id}` | 更新 |
| DELETE | `/api/schedules/{id}` | 削除 |

---

## 4. データモデル（案）

| テーブル | 主な項目 |
|----------|----------|
| `schedules` | id, user_id, title, start_at, end_at, all_day, … |

---

## 5. 非機能

- タイムゾーン統一。
- 他ユーザーの予定表示（共有カレンダー）の要否は要件で決定。
- **監査ログ**: 予定の作成・更新・削除について、**誰が・いつ・どの予定に対して・何を変更したか**を記録する。

---

## 6. 依存関係

- **前提**: 認証ユーザーに紐づく予定。
