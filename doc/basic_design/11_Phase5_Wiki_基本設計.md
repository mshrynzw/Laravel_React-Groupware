## 1. 目的・スコープ

社内ドキュメントを Wiki 形式で作成・編集・閲覧する。バージョン管理は任意で段階導入可能とする。

---

## 2. 機能概要

| 項目 | 内容 |
|------|------|
| wiki_pages テーブル | スラッグ、タイトル、本文、更新者、親ページ（階層任意） |
| 編集 API | 作成・更新、競合制御 |
| バージョン管理（任意） | 履歴テーブルまたはリビジョン番号 |
| エディタ UI | Markdown／WYSIWYG は実装で選択 |

---

## 3. 画面・API（概要）

### 3.1 画面

- Wiki 一覧・階層ナビ
- ページ表示・編集

### 3.2 API（例）

| メソッド | パス（案） | 説明 |
|----------|------------|------|
| GET | `/api/wiki/pages` | 一覧 |
| GET | `/api/wiki/pages/{slug}` | 取得 |
| POST | `/api/wiki/pages` | 作成 |
| PUT | `/api/wiki/pages/{slug}` | 更新 |

---

## 4. データモデル（案）

| テーブル | 主な項目 |
|----------|----------|
| `wiki_pages` | id, slug, title, body, parent_id（任意）, updated_by, … |
| `wiki_revisions`（任意） | page_id, body, version, created_at, … |

---

## 5. 非機能

- XSS 対策（HTML 保存時）。
- 全文検索は社内検索ステップと役割分担。
- **監査ログ**: ページの作成・更新・削除およびリビジョン保存時について、**誰が・いつ・どのページに対して・何を変更したか**を記録する（`wiki_revisions` と併せて監査可能とする）。

---

## 6. 依存関係

- **前提**: 認証・編集権限ロール。
