## 1. 目的・スコープ

複数の業務テーブル（お知らせ、Wiki、タスク等）を横断してキーワード検索できるようにする。インデックス設計と検索 API・UI を対象とする。

---

## 2. 機能概要

| 項目 | 内容 |
|------|------|
| 全テーブル横断検索設計 | 対象エンティティの列挙、統一レスポンス形式 |
| インデックス設計 | PostgreSQL 全文検索、または Elasticsearch 等は規模次第 |
| 検索 API | クエリ、ページング、ハイライト任意 |
| 検索 UI | グローバル検索バー、結果一覧（種別フィルタ） |

## 4. データモデル・インデックス

- 各対象テーブルの `tsvector` 列＋ GIN インデックス、または検索用サマリテーブル／外部エンジン。
- 実装方針は性能要件と運用コストで決定。

---

## 5. 非機能

- 権限：検索結果に表示してよいデータのみ返す（ユーザー別フィルタ）。
- タイムアウト・最大件数。
- **監査ログ**: インデックス再構築・検索基盤の設定変更など、運用上重要な管理者操作について、**誰が・いつ・何を実行したか**を記録する（一般ユーザーの検索キーワードを監査ログに残すかは個人情報・運用方針で別途決定する）。

---

## 6. 依存関係

- **前提**: 各機能のデータが存在すること。
- **連携**: Wiki・お知らせ等の本文検索。
