## 3. 共通レスポンス形式

### 3.1 成功（単体リソース）

```json
{
  "data": { }
}
```

### 3.2 成功（一覧・ページング）

```json
{
  "data": [ ],
  "meta": {
    "current_page": 1,
    "per_page": 15,
    "total": 100,
    "last_page": 7
  }
}
```

※ cursor 方式にする場合は `meta.next_cursor` を定義し、クエリ `cursor` で受け取る。

### 3.3 失敗

| HTTP | 用途 |
|------|------|
| 401 | 未認証（トークン欠如・無効） |
| 403 | 認可失敗（ポリシー） |
| 404 | リソース不存在 |
| 422 | バリデーションエラー |
| 429 | レート制限 |
| 500 | サーバ内部エラー（本番では詳細を隠す） |

```json
{
  "message": "人が読む概要",
  "errors": {
    "field_name": ["メッセージ1"]
  }
}
```

業務コードが必要な場合のみ `code` を追加（例: `"code": "PAYROLL_ALREADY_CLOSED"`）。

## 4. リクエスト共通

| 項目 | 方針 |
|------|------|
| Content-Type | `application/json`（ファイル API を除く） |
| 認証 | `Authorization: Bearer {token}`（SPA + Sanctum トークン想定）またはセッション Cookie（同一構成時） |
| 相関 ID | `X-Request-Id` をクライアント生成またはサーバ付与し、ログに出力 |

---

## 5. Laravel 実装マッピング

| 関心事 | 配置例 |
|--------|--------|
| 入力検証 | `App\Http\Requests\...` |
| 認可 | `App\Policies\...` + `authorize()` |
| ドメインロジック | `App\Services\...`（肥大化したら分割） |
| 永続化 | Eloquent モデル + リポジトリは必要時のみ |

---

## 6. ログ

| レベル | 例 |
|--------|-----|
| info | 認証成功、バッチ開始 |
| warning | リトライ可能な外部失敗 |
| error | 例外、5xx 原因 |

個人情報・トークンはマスク。監査対象操作（給与確定、権限変更等）は `audit` チャンネルまたは専用テーブルへ。

---

## 7. フロント（React）共通

| 項目 | 方針 |
|------|------|
| HTTP クライアント | `fetch` または axios いずれかに統一、401 時はログインへ誘導 |
| エラー表示 | API の `message` / `errors` をフォーム・トーストに反映 |
| ルート保護 | 認証必須ルートをラップするコンポーネント（例: `ProtectedRoute`） |

## 8. テスト観点（優先度低でも記録）

| 種別 | 対象 |
|------|------|
| 単体 | Service の分岐、集計ロジック |
| API | 認証・バリデーション・主要ステータスコード |
| E2E | ログイン〜主要画面のスモーク |

---

## 9. データベース共通規約

- 本プロジェクトで定義する **全テーブル** に、次の行を **必須（NOT NULL）** とする。

`| created_at / updated_at | timestamp | NO | |`

- Laravel ではマイグレーションで `$table->timestamps();` を付与し、DB 制約上も NOT NULL とする（`nullableTimestamps()` は使用しない）。
